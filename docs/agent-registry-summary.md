## Write-Capable Agents

Only these agents have `tool_requests` write access enabled:

### OKR Coach (`okr_coach.json`)

- **All tools:** docs.retrieve, okrs.status, okrs.create, okrs.update_kr
- **Prompt:** `packages/agents/prompts/okr_coach.md`

### Process Mapper (`process_mapper.json`)
- **Purpose:** Interviews stakeholders to extract, normalize, and visualize business processes as structured data and Mermaid diagrams
- **Write tools:** process.create, process.update
- **All tools:** memory.search, process.create, process.update
- **Prompt:** `packages/agents/prompts/process_mapper.md`

### Slack Notifier (`slack_notifier.json`)
- **Purpose:** Sends Slack notifications for approvals, task assignments, OKR updates, and weekly digests. Posts to channels or sends direct messages when Friday takes actions that require human awareness.
- **Write tools:** slack.post, notifications.create
- **All tools:** slack.post, slack.dm, notifications.create
- **Prompt:** `packages/agents/prompts/slack_notifier.md`

## Adding a New Agent

```bash
python scripts/create_agent_from_template.py
```

Then:
1. Edit `packages/agents/manifests/<id>.json` — set purpose, tools_allowed, governance
2. Edit `packages/agents/prompts/<id>.md` — write the system prompt
3. If write tools needed: add to `tools_allowed` and add `tool_requests` instructions to prompt
4. Run `python scripts/update_repo_map.py` to update this file

## Governance Notes

- All agent outputs pass through `PolicyEngine` before write execution
- High-risk writes trigger `ApprovalService` before `ToolExecutor.run()`
- `AuditLog` records every run with full `RunTrace` (inputs, memos, synthesis, write_actions)
- `ReflectionWorker` generates lesson candidates from each `RunTrace`
